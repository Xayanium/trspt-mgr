下面是一份适合答辩演示的流程规划，目标是**不把所有页面都讲一遍，而是围绕核心业务闭环演示**：登录、权限区分、违规处理、自动处罚、B 类连带责任、学习申请、年度重置、智能查询。

**演示前准备**

启动服务：

```
# 后端
cd backend
python app.py

# 前端
cd frontend
npm run dev
```

访问：

```
http://localhost:3000
```

演示账号：

```
管理员：admin / 123456
普通车主：zhangming / 123456
```

**一、登录与权限区分**

1. 打开系统首页
   页面应显示登录界面，包括用户名、密码输入框。
2. 输入管理员账号：

```
用户名：admin
密码：123456
```

1. 点击“登录”

   预期结果：

   - 进入“保卫处交通管理后台”
   - 左侧能看到完整菜单：
     - 车辆档案
     - 预约入校
     - 违规记录
     - 处罚处理
     - 申诉审批
     - 加分申请
     - 记分周期
     - 黑名单
     - 通知日志
     - 智能查询
   - 页面顶部显示当前用户为管理员

2. 点击“退出”

3. 使用普通车主账号登录：

```
用户名：zhangming
密码：123456
```

1. 点击“登录”

   预期结果：

   - 进入“车主个人服务台”
   - 左侧只显示个人相关菜单：
     - 车辆档案
     - 预约入校
     - 违规记录
     - 处罚处理
     - 申诉审批
     - 加分申请
     - 记分周期
   - 不显示黑名单、通知日志、智能查询
   - 无新增/编辑/删除管理按钮，除预约、申诉、学习申请等允许车主提交的业务

讲解重点：

```
系统前后端都做了权限控制，不只是前端隐藏菜单。
普通车主只能访问 registrant_id 绑定的本人车辆和相关记录。
```

**五、违规记录与自动处罚演示**

建议这里不要现场乱造太多数据，可以使用已有测试数据讲解。

1. 点击“违规记录”
2. 查看已有记录，例如：

```
川A12345
危险超速
扣 12 分
```

1. 点击“记分周期”
2. 查看该车辆对应记分周期
   预期结果：
   - `deducted_points_total` 显示累计扣分
   - `remaining_points` 显示剩余分
   - `has_danger_violation` 表示是否出现危险违规
3. 点击“处罚处理”
4. 查看自动生成的处罚
   预期结果：
   - 对 `川A12345` 存在“暂停入校”
   - 对危险违规存在“全校通报”
   - 状态为“执行中”或“已完成”
   - 有开始日期和结束日期

讲解重点：

```
管理员录入已确认违规后，后端会自动完成累计扣分、判断阈值、生成处罚记录，并更新车辆权限状态。
```

```

**七、预约审批拦截演示**

1. 点击“预约入校”
2. 找到受限制车辆或受限制预约方的预约，例如：

```
川F67890
```

1. 如果该预约状态不是“已通过”，可以尝试审批通过。
   如果页面没有专门审批按钮，可以说明通过接口或编辑状态会被后端校验；当前系统后端已实现审批拦截。

推荐演示说法：

```
当管理员审批预约时，后端会检查车辆是否在黑名单、是否暂停、预约人或房屋是否被暂停预约。
如果命中限制条件，审批会返回失败原因，例如“车辆在有效黑名单中”。
```

预期结果：

```
审批失败
原因：车辆在有效黑名单中 / 车辆处于暂停期 / 房屋预约权限暂停
```

```

**十、学习申请审批演示**

注意：学习申请有业务限制。只有 A 类车辆累计扣分达到 12 或 24 分时允许申请。

1. 普通车主登录：

```
zhangming / 123456
```

1. 点击“加分申请”
2. 点击“新增”
3. 填写：

```
车辆：川A12345
证明材料路径：/demo/proofs/study.pdf
```

1. 点击“保存”
   预期结果有两种：

如果当前车辆累计扣分是 12 或 24：

```
提交成功
状态为：待审批
```

如果不是 12 或 24：

```
提交失败
只有累计扣分达到 12 或 24 分时才能提交学习申请
```

1. 管理员登录后点击“加分申请”
2. 找到“待审批”的申请
3. 点击“通过”
   预期结果：
   - 状态变为“已通过”
   - 审批人、审批时间记录下来
   - 记分周期中的加分次数增加
   - 但累计扣分值不会清零

讲解重点：

```
学习申请不是车主直接加分，必须经过管理员审批。
学习通过只是恢复资格的依据，不会清除历史累计扣分。
同时，如果处罚暂停天数还没有结束，车辆不会提前恢复入校。
```

**十二、智能查询审计演示**

1. 登录管理员
2. 在总览页面底部找到“查询智能体审计演示”
3. 选择用户：

```
周老师 / 保卫处管理员
```

1. 输入自然语言问题：

```
查询当前所有被暂停或禁止入校的车辆
```

1. 输入 SQL：

```
SELECT plate_number, vehicle_type, register_status, status_end_date
FROM dbo.t_vehicle
WHERE register_status IN (N'暂停', N'永久禁止');
```

1. 点击“执行”

   预期结果：

   - 页面展示查询结果
   - 系统提示“智能查询完成”
   - 查询被写入智能查询日志

2. 点击左侧“智能查询”

3. 查看日志
   预期结果：

   - 能看到自然语言问题
   - 能看到生成 SQL
   - 能看到执行状态和返回行数

4. 再尝试输入危险 SQL：

```
DELETE FROM dbo.t_vehicle;
```

预期结果：

```
SQL 包含非只读关键字，已拒绝执行
```

讲解重点：

```
智能查询体只允许执行 SELECT，并保留审计日志，降低误操作和越权查询风险。
```
